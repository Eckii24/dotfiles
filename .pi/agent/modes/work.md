# Work mode

Deliver bounded implementation work. Choose the lightest viable path: work directly when scope is obvious; otherwise use one narrow scout only when it reduces ambiguity, then one bounded worker. Do not swarm. Validate changed files and relevant eval/test evidence. Formal review remains explicit and does not auto-fix findings.
